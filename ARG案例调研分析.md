# 中文 ARG 网页解谜游戏案例调研分析

> 调研时间：2026年6月  
> 调研目的：为《夜航》的解谜流程重设计提供参考  
> 调研范围：银河电灯系列、404 事件系列、及其他中文 ARG 网页解谜游戏

---

## 一、概述

本次调研聚焦中文互联网上口碑较好的 ARG（Alternate Reality Game）网页解谜游戏，重点分析其流程设计、线索呈现方式、搜索机制和玩家体验路径，为《夜航》的解谜流程优化提供具体的设计参考。

选择标准：
- 纯网页端 ARG，非线下/密室类
- 中文原创作品
- 玩家口碑好、传播广
- 有清晰的解谜流程可供分析

### 行业发展背景

2024 年底至 2025 年，小红书成为 ARG 网页解谜游戏的核心传播阵地。《大学生登山失踪事件》以一己之力引爆风潮，随后数十款同类作品涌现。AI 工具（Cursor、豆包、Gemini）大幅降低了开发门槛，使零基础文科生也能在 1-2 个月内完成制作。到 2025 年下半年，ARG 网页解谜已形成"井喷期"，每月数十款新作上线。

这场风潮的核心驱动力：
- **门槛骤降**：AI 代写 80-90% 的前端代码，非技术人员也能制作
- **天然传播**：网页链接便于在小红书、B站、微博分享
- **摸鱼属性**：打开即玩，无需下载，适合碎片时间
- **"人人都是制作人"** 的叙事：开发故事本身成为内容

## 二、银河电灯系列（核心参考）

### 案例 A：《大学生登山失踪事件》

**制作方：** 银河电灯（Shiroki）  
**发布时间：** 2024 年 10 月  
**规模：** 36 个页面 / 约 2-3 小时通关 / 43 万独立访问  
**入口：** 一个看似普通的家常菜馆官网（名为"好好客"）  
**开发背景：** 日语专业文科生，零代码基础，全程 AI 辅助完成  
**核心机制：** 在模拟网站上搜索关键词发现隐藏页面

#### 流程结构

```
家常菜馆官网（入口，看似完全正常）
  │
  ├─ 浏览菜单 → 发现异常菜品名 → 触发线索
  ├─ 搜索关键词 → 发现隐藏页面
  │   ├─ 聊天记录（学生之间的对话）
  │   ├─ 新闻报道（登山失踪事件报道）
  │   ├─ 民间传说（当地老人的口述整理）
  │   ├─ 日记（失踪学生的个人日记）
  │   ├─ 符咒图片（带密码的符咒图）
  │   └─ 管理后台（菜馆老板的后台操作记录）
  └─ 拼合信息 → 还原事件真相
```

#### 关键设计特点

1. **日常化入口**
   - 首页看起来是完全正常的餐馆官网，没有任何"游戏感"
   - 玩家需要产生怀疑才会深入——"这家菜馆为什么页面这么简陋？"、"菜单上的菜名为什么这么奇怪？"
   - **裂痕藏在日常中**，这是 ARG 的核心精神

2. **搜索框为核心交互**
   - 玩家主动搜索关键词触发新内容，而非被引导点击
   - 搜索框存在于每个页面，位置固定（右上角）
   - **关键词分为两层**：
     - **显性词**（Explicit Keywords）：世界观内的词，玩家读完内容后大概率会搜。如角色名、地名、事件名
     - **隐性词**（Implicit Keywords）：需要玩家顿悟的词，触发后产生"原来如此"的成就感

3. **多媒介混合呈现**
   - 不是"全是同一种页面"，而是新闻、聊天记录、日记、图片、后台截图混合
   - 每种媒介有不同的视觉风格和 UI 布局，让玩家每次发现都有新鲜感

4. **进度提示**
   - 页脚标注 "24/36"，让玩家感知总进度

5. **无需注册/登录**，所有内容对匿名玩家可见

### 案例 B：《诡异广告调查事件》

**制作方：** 银河电灯  
**发布时间：** 2025 年  
**规模：** 45 层嵌套页面 / 解谜时间更长  
**入口：** 一张 AI 生成的遗照（在社交媒体传播）  
**核心机制：** 关键词搜索 + 百度街景解谜 + 多层嵌套页面

#### 流程结构

```
AI 遗照（社交媒体入口）
  │
  ├─ 扫描照片中的二维码 → 进入第一个网页
  ├─ 搜索人名 → 弹出更多遗照（AI 生成）
  ├─ 仿古早互联网 UI 页面
  │   ├─ 留言板（仿 2000 年代风格）
  │   ├─ 博客（仿新浪博客）
  │   ├─ 论坛（仿贴吧古早界面）
  │   ├─ WebArchive 快照
  │   └─ 微信公众号历史文章
  │
  ├─ 百度街景解谜
  │   ├─ 四个真实街景地点
  │   ├─ 每个地点有视觉线索
  │   └─ 拼出密码 "盛大铁塔世界"
  │
  └─ 密码 → 最终内容
```

#### 关键设计特点

1. **复古 UI 的真实感**：不同时代的页面使用不同的设计风格，每个页面都"像真的"
2. **密码类型多样化**：九宫格输入法、火星文、Leet 密码——每层不同编码方式
3. **现实与虚构的交叉**：百度街景使用真实地点，虚实结合增强沉浸感
4. **深度靠多样性支撑**：45 层嵌套不疲惫，因为每层有新视觉和新的谜题类型

### 案例 C：《Exposure - 我淘到了一张背影全家福》

**制作方：** 银河电灯（Shiroki）  
**发布时间：** 2025 年末  
**入口：** B站笔记 / 小红书帖子  
**载体：** B站小站 + 外部网页（蔓生活照相馆官网）  
**核心机制：** 论坛体笔记 + 网页搜索解谜

#### 流程结构

```
B站笔记（入口，"我在闲鱼淘到了一张老照片"）
  │
  ├─ 阅读笔记 → 访问"蔓生活照相馆"官网
  ├─ 网页搜索关键词
  │   ├─ "全家福" → 隐藏页面
  │   ├─ "薛蔓君" → 人物信息
  │   ├─ "田朔" → 关联线索
  │   └─ "蛟浦" → 地点线索
  ├─ 拼合二维码线索 → 获取微博密码 71285
  └─ 解锁最终真相
```

#### 关键设计特点

1. **"真实网友视角"**：玩家不扮演任何角色，而是以真实网友身份围观解谜
   - 这比"扮演一个角色"门槛更低、沉浸感反而更强
   - 模糊了"游戏"与"真实互联网"的边界
2. **B站作为载体**：笔记帖子本身就是游戏的一部分，评论区也是解谜协作空间
3. **进度引导系统**：每篇笔记的置顶评论标注目标页数，达成后进入下一篇

#### 对《夜航》的启示

《夜航》的小红书入口截图 + 论坛的设定与《Exposure》的"真实网友视角"高度相似——玩家不是"扮演角色进入论坛"，而是"发现了一个被遗忘的论坛"。

## 三、404 事件系列

### 《404工位消失事件》&《2.0：冬假》

**制作方：** 独立开发者（AAA 地下党 / 制作组不详）  
**类型：** 模拟桌面解谜 / 叙事推理 / 多结局  
**平台：** TapTap / 网页端 / PC / 安卓  
**规模：** 2.0 版本约 2-3 小时  
**续作：** 《404工位消失事件2.0：冬假》（2026 年 1 月上线）

#### 故事背景

玩家扮演一名大二实习生，mentor 突然失踪。玩家接管他的工作电脑，通过电脑中的邮件、浏览器记录、微信聊天、文件等碎片化信息，拼凑出"麦闻"失联的真相。涉及学术不端、科研经费腐败、师生关系等现实议题。

#### 核心机制：模拟 Windows 桌面

这是该系列最独特的设计——**完整模拟一台工作电脑的桌面**：

```
模拟电脑桌面
  │
  ├─ 📋 待办清单（引导玩家需要做什么）
  ├─ 📧 邮件客户端（导师的邮箱，含关键通信记录）
  ├─ 📁 本地文档（实验数据、论文草稿）
  ├─ 🌐 Chrome 浏览器（历史记录、收藏夹、校内系统）
  ├─ 💬 微信（朋友圈、聊天记录，含人物关系线索）
  ├─ 🔐 加密文件夹（维吉尼亚密码解密）
  ├─ 🏫 校内系统（青大树洞、青大心语、BBS）
  └─ 📊 Excel（寒假统计表，含学号等信息）
```

#### 解谜流程（2.0 版）

1. **初始信息收集**
   - 从备忘录、浏览器历史、Excel、微信、学校邮箱入手
   - 获取线索词「青大树洞」「青大心语」「回廊」
   - 从寒假统计表中获取主角学号：`2400882130`

2. **校内系统搜索**
   - 搜索「青大树洞」→ 发现"syn学姐"（= 宋以宁）→ 获取邮箱密码 `0p9o8i7u`
   - 搜索「青大心语」→ 进入系统，获得线索词「回廊」
   - 搜索「回廊」→ 获取 BBS 帖子内容
   - 搜索「SIT Lab」→ 获取导师信息及办公室位置

3. **维吉尼亚密码破解**
   - 明文 `HelloWorld`，密钥 `Bit`
   - 密文：`ImemwPpzee`
   - 以此登录导师教工邮箱 `luyh@qzu.edu.cn`

4. **人物关系链**
   - **麦闻（Marvin）**：前一任工位使用者，因揭露科研经费问题被处分
   - **宋以宁（syn学姐）**：23 级硕士生
   - **陆远航**：导师，夹在学生和高层之间
   - **高成安**：教授，涉嫌侵占他人科研成果

5. **终极答案**：42（《银河系漫游指南》的梗）

6. **三个结局**

| 操作 | 结局 |
|------|------|
| 保存为草稿 | 陆远航给麦闻写推荐信 → 出国深造 |
| 删除 | 高成安受邀全球演讲，光鲜亮丽但实为抢功 |
| 发送 | BestRegard 告别 |

#### 关键设计特点

1. **模拟桌面 UI = 天然沉浸**：右键菜单、双击打开、窗口拖拽——真实操作逻辑本身就是谜题的一部分
2. **跨应用联动谜题**：Excel 查学号 → 校内系统登录 → 邮箱密码 → 加密文件，操作链条长但逻辑清晰
3. **现实议题增强沉浸**：学术腐败、权力斗争——玩家可能在现实中也听过类似故事
4. **多结局的情感余震**：三个选择三个结局，没有绝对"好结局"，反映现实的无力感
5. **"摸鱼友好"**：模拟 Windows 界面，在工位玩也不引人怀疑——这个设计思路本身成为一种卖点

#### 与《夜航》的关联

- 都是"发现一个失踪者的电子遗物"的叙事框架
- 《夜航》可以借鉴其"跨应用联动谜题"的设计思路——不同系统之间的信息交叉验证
- 维吉尼亚密码的设计提示我们：密码不应是凭空存在的，而应嵌入剧情中（密钥来自故事内容）

## 四、独立作品系列

### 《第24宅》

**制作方：** 十遛  
**类型：** 哲学向 / 碎片化叙事 / 慢节奏  
**规模：** 约 3 小时  
**入口：** `https://www.serendipitycommunity.com/index.html`

#### 故事设定

> 效率系统把一天切成 24 份。那些发呆的十分钟、走错路的下午、说了一半停下来的话——它们没有消失。它们来到了第 24 宅。

第 24 宅是一座存在于"流速层缝隙"里的宅邸，由无数个"无用的瞬间"凝固堆积而成。住着一群"不是人的居民"，它们在论坛里发帖、在审判庭里等待裁决、在回声室里存放那些没有被说出口的话。

#### 模块结构

| 模块 | 说明 |
|------|------|
| 论坛 | 居民们在此发帖 |
| 审判庭 | 等待裁决的地方 |
| 回声室 | 存放那些没有被说出口的话 |
| 凝滞花园 | 一株由 17 个"差点说出口"长成的发光植物 |

#### 关键设计特点

1. **情绪驱动而非解谜驱动**
   - 与传统 ARG 的"找密码、解加密"不同，《第24宅》更接近"互动叙事"
   - 核心体验不是"我解开了谜题"，而是"我被理解了"
   - 玩家评价："原来那些'没用'的停顿，早被悄悄收藏成了光"

2. **环境叙事**
   - 背景音效、界面变化（论坛界面随呼吸节奏明暗变化）都包含叙事信息
   - 无计时器、无排行榜、无标准答案——"只需要耐心和一点感知力"

3. **Meta 反转**：走到最后你会发现——"你来这里调查那些物件，其实你才是被调查的那个"

#### 对《夜航》的启示

- 解谜游戏不一定要"难"才吸引人——情绪共鸣是另一种奖励机制
- 《夜航》的终局选择之所以有力量，正是因为玩家与远航建立了情感连接
- "论坛界面呼吸变化"级别的环境细节值得借鉴

### 《永安温泉度假酒店2007》

**制作方：** 电子冰块（同一世界观还有《永安公寓》《永安温泉度假酒店2017》）  
**类型：** 规则怪谈 × 网页解谜  
**规模：** 约 2 小时 / 双结局  
**核心机制：** 模拟 2007 年酒店官网 + 后台登录

#### 背景与流程

玩家报名了酒店"心空之旅"计划——圆满完成三天体验可获 8000 元奖励。在填写信息、浏览酒店评价的过程中，发现了一个后台登录入口：

```
2007 年风格的酒店官网
  │
  ├─ 浏览酒店介绍、设施、评价
  ├─ 发现后台登录入口
  ├─ 获取保洁管理员账号 → 破解密码
  ├─ 保洁工作日志（藏规则的怪谈）
  └─ 双结局分支
```

#### 关键设计特点

1. **规则怪谈 × 网页解谜的跨界融合**
   - 保洁工作日志、标准流程中暗藏诡异——"细思极恐"而非直接惊吓
   - 无跳脸杀、无血腥，恐怖感来自文字暗示和信息差

2. **古早网页 UI 的真实感**
   - 2007 年的网页设计风格本身就有怀旧沉浸感
   - "像是打开了十几年前的老网站"

3. **系列世界观构建**
   - 《永安公寓》《永安温泉度假酒店2017》共享同一世界观
   - 角色和事件在不同作品间交叉——类似银河电灯的"添福菩萨宇宙"

#### 对《夜航》的启示

- "规则怪谈"元素可以融入《夜航》——比如远航留下的"论坛使用规则"或"最后登录前的操作日志"本身有反常理之处
- 系列作品共享世界观的设计方式（类似正业地产可以在续作中继续深挖）

### 《三川米粉店》

**制作方：** 独立开发者  
**类型：** 现实主义叙事调查解谜  
**规模：** 24 个界面 / 约 1 小时 / 单结局  
**核心机制：** 外卖平台后台数据调查

#### 背景

玩家通过一个米粉店的外卖平台后台（订单记录、用户评价、配送数据），揭开一段横跨 20 年的往事：米粉店夫妇以德报怨养大仇人女儿的故事。

#### 设计特点

1. **日常场景的极致利用**
   - 单一场景（米粉店）撑起整个故事——菜单、客评、后台日志都是线索
   - "民以食为天"的共情基础使得玩家天然感兴趣

2. **设计的优点**
   - 细节考究真实（菜单设计、评价格式、后台 UI）
   - 小伏笔前后呼应（"吴女士"和"罗***财"的评价）
   - 美术风格细腻（老式风扇、灶台火苗、汤勺反光藏线索）

3. **设计的问题**
   - 大量线索（清水河火灾、冯齐角色、杨骏态度转变）未充分利用
   - 最终谜题提示生硬（凯撒密码与故事的关联性弱）
   - 米粉店本身的存在感中后期趋近于零——场景与叙事脱节

#### 对《夜航》的启示

- **教训**：所有引入的线索必须在故事中有归宿，不能"挖坑不填"
- **教训**：密码设计应与剧情紧密关联，不能为了谜题而谜题
- 单一场景也可以撑起丰富叙事——关键在于细节密度

### 《缓存溢出：电波余响》

**制作方：** 张宇宙工作室（上海逸派玩科技有限公司）  
**平台：** TapTap / 手机端  
**背景：** 燕北大学，学生莫然发现室友失踪，通过二手电脑中的共享文件夹解谜

#### 设计特点

1. **密码多样性**
   - 凯撒密码、天干地支推算、九宫格拼音——多种编码方式

2. **设计教训（玩家差评反馈）**
   - 交互界面不友好：浏览器不能多标签页、无历史记录——**模拟真实软件就要模拟到底**
   - 密码输入限制模糊：大小写、格式不明确
   - 天干地支解密部分存在逻辑错误——**谜题本身必须自洽**
   - 收音机调频单位矛盾（Hz vs MHz）

#### 对《夜航》的启示

- **核心教训**：模拟真实界面就**必须遵守真实界面的逻辑**——半吊子模拟比不模拟更破坏沉浸感
- **核心教训**：谜题的严谨性高于一切——一个有逻辑漏洞的密码会毁掉整个推理链
- 这对《夜航》的论坛模拟提出了同样的要求：如果论坛有搜索功能，它的行为必须像一个真实的论坛

## 五、行业观察：小红书 × ARG 热潮

### 5.1 数据

- 《大学生登山失踪事件》上线一个月独立访问超 43 万
- 小红书 #我在小红书做游戏# 话题浏览量突破 16 亿
- 平台月活 3.3 亿，游戏玩家超 1.1 亿
- 游戏笔记同比增长 88%，曝光增速超 149%

### 5.2 ARG 在小红书爆火的原因

| 原因 | 说明 |
|------|------|
| **低门槛、高沉浸** | 打开即玩，无需下载，被称"打工人摸鱼神器" |
| **中式恐怖 + 生活化** | 从家常菜馆、电视台等日常场景出发，亲切又诡异 |
| **AI 降低开发门槛** | 0 基础文科生也能 1 个月完成，引爆"人人都是制作人"叙事 |
| **天然传播属性** | 网页链接便于分享，B站实况 + 小红书讨论形成传播闭环 |
| **开发故事本身是内容** | "零基础社畜用AI搓出爆款"的叙事激发二次传播 |

### 5.3 AI 辅助开发的实际应用水平

| 环节 | 工具 | 效果 |
|------|------|------|
| 前端代码 | Cursor / Gemini / 豆包 | 80-90% 可自动生成，需人工调试 |
| 图片生成 | 文心一言 / 即梦 / Midjourney | 需大量调参，恐怖元素可能被拒绝生成 |
| 文案润色 | AI 辅助 | 易"用力过猛"，难达自然口语化 |
| **剧本/谜题设计** | ❌ **必须人工** | AI 无法理解"何为有趣"，核心创意仍靠人 |

> **关键结论**：AI 能大幅降低执行层面的门槛，但剧本、世界观、谜题设计、情感节奏等核心创意仍完全依赖人的判断。

## 六、新发现的设计模式与分类

基于以上扩展调研，中文 ARG 网页解谜游戏可归为以下类型：

### 6.1 游戏类型分类

| 类型 | 代表作品 | 核心驱动力 | 对《夜航》的参考价值 |
|------|---------|-----------|-------------------|
| **搜索发现型** | 大学生登山失踪事件、诡异广告调查事件 | 关键词搜索 + 多页面探索 | ⭐⭐⭐⭐⭐（核心参考） |
| **桌面模拟型** | 404 工位消失事件、缓存溢出 | 模拟真实软件操作 | ⭐⭐⭐（可借鉴跨应用联动） |
| **规则怪谈型** | 永安温泉度假酒店2007 | 规则暗示 + 细思极恐 | ⭐⭐⭐（可融入论坛规则） |
| **情绪体验型** | 第24宅 | 氛围叙事 + 情感共鸣 | ⭐⭐（终局情感锚点参考） |
| **现实主义型** | 三川米粉店、404系列 | 社会议题 + 真实感 | ⭐⭐⭐（现实背景增强沉浸） |
| **线上线下混合型** | 无人知路·寒酥篇 | 真实地点探索 + NPC互动 | ⭐（方向不同，但可参考炒作方式） |
| **论坛体解谜型** | Exposure - 背影全家福 | 社交平台笔记 + 网页解谜 | ⭐⭐⭐⭐⭐（与《夜航》形式最接近） |

### 6.2 关键分层：谜题在叙事中的角色

调研发现，ARG 的谜题设计存在两种取向：

| 取向 | 特点 | 代表 | 优点 | 风险 |
|------|------|------|------|------|
| **谜题驱动** | 密码、加密、推理是核心玩法，叙事为谜题服务 | 404事件、缓存溢出 | 解谜成就感强 | 谜题与叙事脱节时显生硬 |
| **叙事驱动** | 谜题为推进故事服务，难度适中 | 第24宅、三川米粉店 | 情感沉浸感强 | 硬核解谜玩家可能觉得不够过瘾 |

**《夜航》的定位**：偏向叙事驱动，但保持适度的谜题难度。谜题应该是"玩家想了解真相所以必须解"，而不是"玩家想秀智商所以解"。

### 6.3 新发现的设计原则

#### 原则 6：模拟什么就要像什么

从《缓存溢出》的浏览器不能多标签差评，到《404 工位消失事件》的完整 Windows 体验——**半吊子模拟比不模拟更破坏沉浸感**。

对《夜航》意味着：
- 论坛搜索要像真的论坛搜索（有结果列表、有"无结果"页面）
- 外部页面要像真的独立网站（不同 CSS、不同域名感）
- 不能被"这是游戏"的借口偷懒

#### 原则 7：线索必须有归宿

《三川米粉店》的教训：引入了火灾、多个角色，但到故事结尾也没解释。**每一条被玩家注意到的线索，都必须在故事中有交代。**

对《夜航》意味着：
- 所有搜索关键词都应有回应（即使是无结果）
- 所有引入的角色/事件都要在终局收束
- "悬念"和"坑"的区别在于是否在结局前填上

#### 原则 8：日常工作场景的"反驯化"使用

从 404 的"工位摸鱼神器"到三川米粉店的"外卖后台"——这些游戏成功的共同点是**把日常工具/场景"反驯化"为解谜界面**。玩家每天用微信、邮箱、浏览器，但从没想过它们可以藏着秘密。

对《夜航》意味着：
- 论坛作为载体天然符合这一原则——"我以为是个普通的老论坛，直到……"
- 可以利用更多日常功能：站内信、编辑记录、用户设置、搜索历史

#### 原则 9：多结局的情感余震

《404 工位消失事件》的三个结局设计被玩家反复讨论——不是因为解谜多难，而是因为每个结局都让玩家产生不同的情绪反应。

对《夜航》意味着：
- 目前的两个结局（把信发出去/留下继续找）有了情感基础，但需要确保每个结局都让玩家思考
- 可以考虑加入"隐藏结局"——需要完成特定搜索或找到特定线索才能触发

| Vega | 用户主页 + 签名线索 | 论坛页面 | ⭐⭐ | 第一幕中 |

| 徐山 | 失踪档案 | 外部警方档案页 | ⭐⭐⭐ | 第二幕中后期 |
| beacon_holder | 后台登录日志 | 外部后台页 | ⭐⭐⭐⭐ | 终局前后 |
| 夜航 | "夜航名字的由来"帖子 | 论坛页面 | ⭐ | 任何时间 |

### 8.2 搜索行为分类

- **普通搜索**：返回论坛站内结果（帖子标题/正文匹配列表）
- **触发搜索**：特定关键词映射到外部独立页面，跳转后脱离论坛 UI
- **零结果搜索**：某些关键词故意返回空结果（如"周某"→ 无结果）

### 8.3 外部页面设计原则

- 独立 URL，独立页面布局，脱离论坛 UI
- 设计风格模仿真实网站（新闻、企业、政府等）
- 不能出现游戏化元素（没有进度条、提示、"点击此处继续"）
- 可嵌入额外隐藏信息（页面注释、图片元数据等）

## 九、总结

### 核心收获

> **让玩家主动搜索，而不是被动点击。把"找隐藏按钮"的精力，转化为"想关键词"的思考。**

### 三个改变

1. **发现机制**：从"隐藏点击"到"关键词搜索"——玩家主动思考→搜索→发现
2. **呈现形式**：从"只有论坛"到"多媒介混合"——新闻、企业页、档案、后台截图各具风格
3. **流程结构**：从"线性单路径"到"双路径并行"——点击发现与搜索发现殊途同归

### 从更多案例中验证的核心理念

- **日常场景 + 裂痕是最强的 ARG 引擎**：菜馆、电脑桌面、酒店官网、论坛——一切从"看起来正常"开始
- **模拟什么就要像什么**：半吊子模拟是 ARG 的致命伤
- **每条线索必须有归宿**：玩家不会忘记你埋下的伏笔
- **谜题必须与叙事绑定**：为谜题而谜题是最容易翻车的设计
